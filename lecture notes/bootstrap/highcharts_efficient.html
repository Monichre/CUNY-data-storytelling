Highcharts

This template will show you how to place a Highcharts chart in an HTML page. No iFrames needed

By Amanda Hickman | @amandabee | Mar 31, 2014

We haven't talked about it much, but if you look at your charts, you'll see that the JSFiddle HTML pane includes <div id="container"...> while the JavaScript pane instantiates a chart with:

$(function () { $('#container').highcharts({'

#container refers to the HTML element with id "container" -- in this case, your div. You used the same name ("container") to set the class of the div that contains all of your content. A class is different from an id, so they won't conflict, but it is worth giving your charts more intelligible names anyway.

Classes and Ids

What you need to know: you can reuse a class over and over on a page. You can't reuse an id. You can see why when you start to deal with HighCharts: the javascript needs some space on the page to work in and it needs to know exactly which div to draw the chart in.

Where do scripts go?

I used JSCompress to "minify" my javascript and put it at the bottom of the page. The change should help everything load more effectively. At least ... that's what the experts say.
